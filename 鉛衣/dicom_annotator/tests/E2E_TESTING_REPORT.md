## 4. 缺陷分類簡化優化驗證

根據使用者需求，系統的缺陷類別已簡化為以下兩種：
1. **破洞** (圓形型/預設，對應 `hole`/`void`/`defect`)，顯示顏色：**藍色 (`#40c4ff`)**。
2. **裂痕** (線條型，對應 `crack`)，顯示顏色：**紅色 (`#ff5252`)**。

*   **相容性設計**：歷史儲存的舊標註類型（如 `defect`, `void` 等）已自動映射為「破洞」並正確顯示對應顏色；匯出 YOLO/COCO 時，歷史類別亦已與新類別一同在後端完成標準化對應（`hole`/`void`/`defect` $\rightarrow$ Class `0`；`crack` $\rightarrow$ Class `1`）。

## 5. 多選刪除功能 (Batch Deletion) 容錯修復驗證

針對使用者按住 Shift/Ctrl 多選時刪除檔案無反應的問題，已完成全面檢修：
*   **根本原因**：舊資料庫中既存的 DICOM 路徑多為絕對路徑，而刪除 API 在處理前端傳遞過來的路徑時，僅以標準化後的相對路徑（`norm_path`）查詢資料庫，導致資料庫判定「找不到該路徑檔案記錄」而直接略過刪除。此外，錯誤回報中存有未定義的變數 `pstr` (NameError 崩潰隱患) 亦已修正。
*   **修復做法**：後端刪除 API 已升級為「三層式容錯查庫」，會同時比對相對路徑、解析後的本機絕對路徑與原始字串。確保不論資料庫是存何種路徑格式，皆能精準對應、成功將其實體檔案與資料庫紀錄徹底清除。
*   **驗證結果**：E2E 自動化測試再次執行且 100% 成功通過，證明單選及多選刪除工作流功能已可正常完美執行。
