# Med-Research-Skill & Deep Research Knowledge Base (v10.0 Master)

## 🌟 Project Overview
本專案為高階醫學研究與自動化深度挖掘知識庫，整合了臨床實戰規範與 AI 研究協議。適用於各種臨床醫學、影像醫學、公共衛生及醫療管理領域的深度研究。

Born in a clinical environment, this skill ensures that every research step adheres to the highest academic standards (JCR Q1/Q2) and follows a systematic protocol for evidence triangulation.

## 🛠️ Key Features (v10.0 Standard)
- **JCR/IF 強制驗證**: 所有研究報告必須對標 JCR (Q1/Q2) 與實體 Impact Factor (IF)，嚴禁使用非認證文獻。
- **1:1 數據誠信協議**: 強制執行「DOI -> 標題 -> 技術細節」實體驗證，徹底杜絕 AI 幻覺。
- **統一 10 欄位矩陣**: 標準化技術輸出格式，涵蓋臨床挑戰、技術巧思與數據流 Logic。
- **Multi-Source Discovery**: 整合 PubMed (NCBI)、Google Scholar 與跨出版商下載梯隊。
- **Agentic Workflow**: 評估技術文獻的 AI 代理落地方案，支持臨床決策支援系統 (CDSS) 開發。

## 📋 Research Protocol (GARP v10.0)
All research sessions follow the **General Autonomous Research Protocol**:
1. **Initialize**: 定義臨床問題與研究邊界。
2. **Mine (7-Tier Ladder)**：執行跨平台文獻採礦與實體 PDF 獲取。
3. **Scrutinize**: 執行 JCR 門控與數據一致性審計。
4. **Synthesize**: 生成 10 欄位高密度 Master 矩陣與技術剖析。
5. **Project Mapping**: 針對目標臨床環境進行實體對標與發表利基分析。

## 📂 Directory Structure
- `Med Deep Research/reports/`: 存放各版本 Master 報告。
- `Med Deep Research/papers/`: 存放實體 PDF 文獻。
- `med-research/`: Skill 核心代碼與指令集 (`SKILL.md` v10.0)。
- `GEMINI.md`: 項目憲法，定義最高決策權重。

## 🚀 Getting Started
1. **Clone the repo**:
   ```bash
   git clone https://github.com/drew-young-AI/Med-Research-Skill.git
   ```
2. **Install Dependencies**:
   ```bash
   pip install -r med-research/requirements.txt
   ```
3. **Run Analysis**:
   啟動 Gemini CLI 並調用 `med-research` skill 進行深度研究。
